# Routing Rules

## Decide primary route

- Route to frontend skill when UI, interaction, state, router, forms, or component behavior dominates.
- Route to backend skill when domain model, API behavior, data integrity, transaction, or persistence dominates.
- Route to component packaging skill when work must become reusable UI library primitives.
- Route to framework packaging skill when work must become reusable backend starter or platform module.

## Split mixed tasks

1. Extract shared contract and acceptance criteria.
2. Generate parallel subtasks for frontend and backend.
3. Merge through integration checks and end-to-end verification.
